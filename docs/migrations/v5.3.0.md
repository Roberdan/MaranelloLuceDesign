# Migration Guide — v5.3.0

> **Severity:** MINOR
> **Affects:** JS APIs (layout auto-init), CSS bundle (header styles)
> **Migration effort:** Low (one attribute addition)

## Summary

Layout auto-init changed from implicit (any `#mn-grid` in DOM) to explicit (`data-mn-auto-layout` attribute required). This prevents state conflicts for framework consumers (Svelte, React, Next.js) that manage their own slot visibility. Header CSS was missing from `maranello.min.css` — now included. Header buttons emit a new `header-button-click` CustomEvent. `Maranello.sparkline` IIFE export added.

---

## Breaking Changes

### [BC-1] Layout auto-init requires `data-mn-auto-layout` attribute

**Why it changed:** Auto-initializing `createLayout()` on any `#mn-grid` element caused state conflicts with framework consumers. Svelte stores, React useState, and Next.js SSR hydration all fought with Maranello's DOM mutations on load.
**Who is affected:** Anyone using `#mn-grid` with the IIFE bundle who relied on `Maranello.layout` being created automatically.

**Before (v5.2.1):**
```html
<!-- Maranello auto-initialized layout on DOMContentLoaded -->
<div id="mn-grid">
  <div id="mn-slot-strip" hidden></div>
  <div id="mn-slot-left" hidden></div>
  <div id="mn-slot-center"></div>
  <div id="mn-slot-right" hidden></div>
</div>
<script>
  // Maranello.layout was already available
  Maranello.layout.register('dashboard', { label: 'Dashboard' });
</script>
```

**After (v5.3.0):**
```html
<!-- Option A: add data-mn-auto-layout for same auto-init behavior -->
<div id="mn-grid" data-mn-auto-layout>
  <div id="mn-slot-strip" hidden></div>
  <div id="mn-slot-left" hidden></div>
  <div id="mn-slot-center"></div>
  <div id="mn-slot-right" hidden></div>
</div>
<script>
  // Maranello.layout works as before
  Maranello.layout.register('dashboard', { label: 'Dashboard' });
</script>
```

```js
// Option B: explicit init (recommended for framework consumers)
const layout = Maranello.createLayout(document.getElementById('mn-grid'));
layout.register('dashboard', { label: 'Dashboard' });
```

**Migrate:**
1. If you use `#mn-grid` and rely on `Maranello.layout` being auto-created, add `data-mn-auto-layout` to your grid element.
2. If you are a framework consumer (Svelte/React/Next.js), remove `#mn-grid` auto-init reliance and call `Maranello.createLayout(el)` explicitly.

## Non-Breaking Changes

- `header-button-click` CustomEvent — all header buttons now emit this event (bubbles) with `{ id, label }` detail, in addition to the existing `onClick` callback. Enables decoupled event-based wiring for frameworks.
- `Maranello.sparkline` — top-level IIFE export for the sparkline chart factory (was previously only available via `Maranello.charts.sparkline`).
- `components-header.css` — now included in `maranello.css` / `maranello.min.css`. Previously only in `index.css`, causing header to be invisible for consumers using the standard CSS bundle.

## Automated Migration

```bash
# Add data-mn-auto-layout to existing #mn-grid elements
find . -name '*.html' | xargs sed -i '' 's/id="mn-grid"/id="mn-grid" data-mn-auto-layout/g'
```
