# Migration Guide — v6.0.0

> **Severity:** MAJOR
> **Affects:** All consumers

## Summary

The monolite `maranello-luce-design-business` is split into two packages:

- `@maranello/tokens` — CSS variables, 5 themes, setTheme/cycleTheme, shadcn/ui bridge
- `@maranello/elements` — 31 Web Components, headless viz/domain API

The Presentation Runtime (AppShellController, ViewRegistry, NavigationModel, PanelOrchestrator, StateScaffold) has been removed. Consumers should use framework-native equivalents (Next.js App Router, SvelteKit layouts, etc.).

## Install

```bash
# Old
npm install maranello-luce-design-business

# New
npm install @maranello/tokens @maranello/elements
```

## Import Changes

```typescript
// Old
import { setTheme, kanbanBoard, FerrariGauge } from 'maranello-luce-design-business';

// New — tokens
import { setTheme, cycleTheme, palette } from '@maranello/tokens';
import '@maranello/tokens/css';
import '@maranello/tokens/bridge-shadcn'; // for shadcn/ui

// New — elements
import { kanbanBoard, FerrariGauge, gantt } from '@maranello/elements';
import '@maranello/elements/css';

// Web Components
import '@maranello/elements/register-all';   // all 31 WC
import '@maranello/elements/wc/mn-gauge';    // single WC
```

## Removed Components

| Component | Replacement |
|---|---|
| AppShellController | Next.js/SvelteKit layouts |
| ViewRegistry | Framework file-based routing |
| NavigationModel | Framework router |
| PanelOrchestrator | Framework layout system |
| StateScaffold | Framework Suspense/loading states |
| loginScreen | NextAuth/Lucia Auth |
| mn-app-shell WC | Framework layout component |
| mn-state-scaffold WC | Framework loading component |
| mn-login WC | Framework auth page |

## DashboardRenderer

`DashboardRenderer` remains available in `@maranello/elements` but no longer depends on `StateScaffold`. It uses a lightweight `WidgetScaffold` internally. If you relied on `StateScaffold` states propagating through `DashboardRenderer`, add your own loading/error/empty handling at the framework level.

## shadcn/ui Bridge

```css
/* Import after tokens to auto-map Maranello colors to shadcn */
@import '@maranello/tokens/bridge-shadcn';
```

The bridge maps Maranello semantic tokens to shadcn/ui CSS custom properties automatically. No manual token mapping is needed.

## CSS Import Changes

```css
/* Old */
@import 'maranello-luce-design-business/css';

/* New — tokens (variables, themes) */
@import '@maranello/tokens/css';

/* New — element styles (component CSS) */
@import '@maranello/elements/css';

/* New — shadcn bridge (optional) */
@import '@maranello/tokens/bridge-shadcn';
```


## Migration Checklist

| Step | Action |
|---|---|
| 1 | Replace `npm install maranello-luce-design-business` with `npm install @maranello/tokens @maranello/elements` |
| 2 | Update all `import ... from 'maranello-luce-design-business'` to use `@maranello/tokens` or `@maranello/elements` |
| 3 | Update CSS imports from `maranello-luce-design-business/css` to `@maranello/tokens/css` + `@maranello/elements/css` |
| 4 | Remove all references to AppShellController, ViewRegistry, NavigationModel, PanelOrchestrator, StateScaffold |
| 5 | Replace loginScreen / mn-login with framework auth solution |
| 6 | Replace mn-state-scaffold with framework loading/error boundary |
| 7 | Add `@maranello/tokens/bridge-shadcn` if using shadcn/ui |
| 8 | Update CDN links if using IIFE builds |
