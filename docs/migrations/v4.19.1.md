# Migration Guide — v4.19.1

> **Severity:** PATCH (WCAG 2.2 AA compliance fixes)
> **Affects:** CSS tokens, Danger button appearance (Avorio), Theme signal colors
> **Migration effort:** Low — search-replace in most cases

## Summary

v4.19.1 introduces a semantic design audit CI gate and fixes four WCAG AA violations found across
the Avorio, Colorblind, and Sugar themes. All changes are accessibility-driven: colors were
adjusted to meet the 4.5:1 minimum contrast ratio (WCAG SC 1.4.3), and the Avorio danger button
was redesigned to be visually distinct from the primary button. If your UI relies on the exact
hex values of the affected tokens or the filled appearance of the Avorio danger button, you will
see visual changes on upgrade.

---

## Breaking Changes

### [BC-1] New token `--mn-danger-text` — danger button text color

**Why it changed:** `.mn-form-btn--danger` and `.mn-btn--danger` previously used
`color: var(--mn-text)` (dark `#111111`) on a red `#DC0000` background. This gives only
2.65:1 contrast ratio — fails WCAG SC 1.4.3 (minimum 4.5:1).

**Who is affected:** Anyone who styled danger buttons by overriding `color:` and relied on
inheriting `var(--mn-text)`.

**Before (v4.19.0):**
```css
/* internally: */
.mn-form-btn--danger { color: var(--mn-text); }

/* your override worked because --mn-text was the base: */
.my-danger-btn { color: var(--mn-text); }
```

**After (v4.19.1):**
```css
/* internally: */
.mn-form-btn--danger { color: var(--mn-danger-text); } /* defaults to #ffffff */
```

**Migrate:** Replace `color: var(--mn-text)` on danger buttons with `color: var(--mn-danger-text)`.
If you need a custom color, override the token:
```css
:root { --mn-danger-text: your-color; }
```

---

### [BC-2] Avorio theme — danger button is now outline instead of filled

**Why it changed:** In Avorio, `--mn-accent` equals `--mn-error` (both are Ferrari red `#DC0000`).
This made the primary action button and the destructive danger button visually identical —
the semantic meaning of "danger" was lost.

**Who is affected:** Avorio theme users who use `.mn-btn--danger` or `.mn-form-btn--danger`.

**Before (v4.19.0) — Avorio:**
Danger button: solid red fill (same as primary button).

**After (v4.19.1) — Avorio:**
Danger button: outline with red border and red text on transparent background. Primary button
retains the solid red fill.

**Migrate:** No code change required — the distinction is automatic. If your designs relied on
a filled danger button in Avorio, you can restore it with:
```css
body.mn-avorio .mn-btn--danger {
  background: var(--mn-error);
  color: var(--mn-danger-text);
  border-color: transparent;
}
```

---

### [BC-3] Colorblind theme — `--mn-error` darkened from `#D55E00` to `#C94000`

**Why it changed:** Okabe-Ito vermillion `#D55E00` has a mid luminance (L≈0.233) — it achieves
only 3.71:1 with white and 2.65:1 with black. Neither passes WCAG AA (4.5:1 required).
Darkening to `#C94000` (L≈0.161) gives 4.98:1 with white.

**Who is affected:** Anyone in the colorblind theme who hardcoded `#D55E00` or captured
`--mn-error` at load time instead of reading it live via `Maranello.palette()`.

**Before (v4.19.0):**
```css
body.mn-colorblind { --mn-error: #D55E00; }
```

**After (v4.19.1):**
```css
body.mn-colorblind { --mn-error: #C94000; }
```

**Migrate:** If you hardcoded `#D55E00` anywhere in your CSS/JS, replace with `var(--mn-error)`.
Always read live values via `Maranello.palette()` in render functions — never capture at load time.

---

### [BC-4] Sugar theme — `--mn-text-muted` darkened from `#767676` to `#5e5e5e`

**Why it changed:** `#767676` on Sugar surface `#E4E4E8` gives only 3.15:1 — fails WCAG AA.
`#5e5e5e` gives 4.6:1.

**Who is affected:** Sugar theme users who rely on `--mn-text-muted` for secondary labels.
The text will appear slightly darker.

**Before (v4.19.0):**
```css
body.mn-sugar { --mn-text-muted: #767676; }
```

**After (v4.19.1):**
```css
body.mn-sugar { --mn-text-muted: #5e5e5e; }
```

**Migrate:** No action needed unless you hardcoded `#767676`. If so, replace with
`var(--mn-text-muted)`.

## Non-Breaking Changes

- New `--mn-danger-text` semantic token added (defaults to `var(--bianco-puro)` = `#ffffff`).
  Use it in any component that places text on `--mn-error` or danger-colored backgrounds.
- New CI gate: `scripts/check-semantic-design.sh` — runs on every push, audits 6 themes
  (96 contrast pairs) and signal distinctness. Zero P0 findings required for CI to pass.
- Sugar + Sugar-Colorblind cross-theme (`body.mn-sugar.mn-colorblind`) now included in all audits.

## Automated Migration

```bash
# BC-1: Replace danger button color usage
find src -name '*.css' | xargs grep -l 'mn-form-btn--danger\|mn-btn--danger' | \
  xargs sed -i 's/color: var(--mn-text)/color: var(--mn-danger-text)/g'

# BC-3: Replace hardcoded D55E00 (colorblind error)
find src -name '*.css' -o -name '*.ts' -o -name '*.js' | \
  xargs grep -rl '#D55E00\|#d55e00' | \
  xargs sed -i 's/#[Dd]55[Ee]00/var(--mn-error)/g'

# BC-4: Replace hardcoded 767676 (sugar muted text)
find src -name '*.css' | xargs grep -rl '#767676' | \
  xargs sed -i 's/#767676/var(--mn-text-muted)/g'
```
